# Seedance 2.0 场景化 Prompt 模板

主 SKILL.md 是通用结构规范。本文档收录官方场景化模板（文字生成、图片参考、视频参考、视频编辑），按需翻阅。

---

## 1. 文字生成（T2V / I2V / R2V / V2V 均支持）

文字优先用常用字，避免生僻字、特殊符号。

### 1.1 Slogan / 广告语

```
「文字内容」+「出现时机」+「出现位置」+「出现方式」，「文字特征（颜色、风格）」
```

示例：
> 手绘漫画风格，三个人围坐在一起吃`图片1`中的炸鸡，气氛友好愉悦，后画面逐渐模糊，画面中部显示文字"快乐尽在 Seedance"。

### 1.2 字幕

```
画面底部出现字幕，字幕内容为"…"，字幕需与音频节奏完全同步。
```

示例：
> 生成带有画外音的视频。一个深沉、平静的男声说："在宏大的宇宙中，我们的世界不过是一个短暂的瞬间。"场景从夜晚缓慢过渡到黎明。画面底部按照台词出现字幕。

### 1.3 气泡台词

```
「角色」说："…"，角色说话时周围出现气泡，气泡里写着台词。
```

示例：
> `图片1`中的两人穿着运动服在学校的操场跑步，女孩看向男孩，自信地笑着说："We can definitely do it!"说话的角色周边出现气泡，气泡里是对应台词。

## 2. 图片参考

参考素材按上传顺序，用 `图片1` ~ `图片n` 指代。

### 2.1 主体多视角

```
参考/提取/结合 +「图片n」中的「主体」，生成「画面描述」，保持「主体」特征一致。
```

商品多视角：
> 提取`图片1`、`图片2`、`图片3`的相机，把背景换成白色，相机在白色桌子上，镜头特写聚焦相机，然后以相机为主体缓慢旋转，清晰展示正面、侧面、背面。

角色多视角：
> 参考`图片1`、`图片2`、`图片3`中的女子形象，生成她在咖啡店吃蛋糕的画面。

### 2.2 多图组合

```
参考/提取/结合/按照 +「图片n」中的「被参考元素描述」，生成「画面描述」，保持特征一致。
```

Logo 参考：
> 参考`图片2`中的女孩，先用中景展示女孩放飞银色悬浮灯，再镜头拉远展现漫天悬浮灯，画面逐渐模糊，后出现`图片1`的 Logo。

多主体：
> 参考图片中的猫猫和狗狗，在温馨公寓里，狗狗趴着吃狗粮，猫猫走过来碰了碰狗狗，两者依偎。画面采用暖色调。

多元素（角色 + 服装 + 场景 + 标识）：
> 场景设定在`图片4`中的餐厅内。`图片1`里的女孩身着`图片2`中的服装，正在整理柜台。`图片3`中的男孩走上前。`图片5`中的标识始终显示在画面右下角。

分镜图：
> 参考图片中的分镜图，生成打斗场面。各分镜构图按顺序出现。

## 3. 视频参考

视频按上传顺序用 `视频1` ~ `视频n` 指代。

### 3.1 动作参考

```
参考「视频n」的「动作描述」，生成「画面描述」，保持动作细节一致。
```

> 参考`视频1`的人物动作和镜头语言，生成`图片2`和`图片1`的打斗场面。

### 3.2 运镜参考

```
参考「视频n」的「运镜描述」，生成「画面描述」，保持运镜一致。
```

> 参考`视频1`的运镜，做一个科技园区的概念视频，以`图片1`中的高楼为视觉中心，第一视角俯冲。

### 3.3 特效参考

```
参考「视频n」的「特效描述」，生成「画面描述」，保持特效一致。
```

> 参考`视频1`的金色粒子特效，让`图片2`中的人物吹笛子时身边环绕一样的粒子特效。

---

## 4. 视频编辑

### 4.1 元素增 / 删 / 改

```
增加：在「视频n」的「时间位置」+「空间位置」，增加「理想元素描述」。
删除：删除「视频n」中的「被删除元素」，视频其他内容保持不变。
修改：将「视频n」中的「被更换元素描述」，替换为「理想元素描述」。
```

> 增：在`视频1`的台面上添加炸鸡、披萨等小吃。
> 删：清除`视频1`桌面上的其他零件和工具，保持桌面整洁干净。
> 改：将`视频1`中的香水替换成`图片1`中的面霜，动作和运镜不变。

### 4.2 视频延长

```
向前/向后延长「视频n」+「需延长的视频描述」
生成「视频n」之前/之后的内容 +「需延长的视频描述」
```

> 模型自动截取衔接部分，不会重复生成原片段。

> 生成`视频1`之后的内容，迟到的两个男士跑向他们，五个人终于见面，友好聊天。

### 4.3 轨道补齐

```
「视频1」+「过渡画面描述」+ 接「视频2」+「过渡画面描述」+ 接「视频3」
```

**限制：** 最多 3 段视频输入，总时长 ≤ 15 秒。

> `视频1`，树叶落地的瞬间激起金色粒子特效，一阵风吹过，接`视频2`。

---

## 5. 与主规范的关系

写场景化 prompt 时仍要遵守主 `SKILL.md` 的硬约束：

1. 动作减速规则照样适用（参考视频里如果是快速动作，描述时也要软化）
2. 含人脸时仍要加约束词
3. 除非用户明确要求文字/字幕，默认加 "禁止出现任何文字、字幕、标题、水印"
4. 多模态指代用 `图片n`/`视频n`，不要用"这张图"
